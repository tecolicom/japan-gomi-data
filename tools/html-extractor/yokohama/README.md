# 横浜市 HTML 抽出

横浜市公式「ごみと資源の収集曜日」(18区・五十音別 126 サブページ) から収集日程を抽出する。
政令市 2 都市目。コースは区ごとにグルーピングし course slug を `<区romaji>-<n>` とする (川崎方式)。

## 手順
1. `node fetch.mjs` — 18 区の index からサブページを発見し 126 ページを cache/ に取得。
2. `node --test parse.test.mjs` — パース純粋関数のテスト。
3. `node build.mjs` — cache/ から municipalities/kanagawa/yokohama/2026/ を生成 (course-<区>-<n>.yaml)。
4. リポジトリ root で `npm test` / `npm run build:ics` で検証。

## 収集体系 (公式列 → 正典カテゴリ)

表のセル語彙は 3 種 + なし の閉集合。ただし同日別袋の品目があり、計 9 カテゴリになる:

- 燃やすごみ (週2) → `burnable`。同じ曜日に別袋で:
  - 燃えないごみ → `non_burnable` (週2、「燃やすごみの収集日と同じ」)
  - 電池類 → `hazardous` (同上)
  - スプレー缶 → `spray_can` (同上)
- 缶・びん・ペットボトル (週1) → `beverage_can`+`glass_bottle`+`pet_bottle` (同日、YAML アンカー共有)。同じ曜日に別袋で:
  - 小さな金属類 → `metal` (週1、「缶・びん・ペットボトルの収集日と同じ」)
- プラスチック資源 (週1) → `plastic`
- 粗大ごみは事前申込制のため rules に入れない。古紙・古布は資源集団回収 (市収集外) のため対象外。

同日別袋品目の頻度の一次ソース: 市「市で収集するもの」各品目ページ (das11=燃えないごみ,
das4=電池類, das10=スプレー缶, das8=小さな金属類)。

## メモ
- **yomi は公式表の五十音列マーカ (初字)** を前方補完して付与。areas は五十音順に整列
  (サブページのファイル名順は五十音順でないため build 時に並べ直す)。
- 区をまたぐ同名町名は `吉田町（中区）` のように区名で曖昧性解消 (2026年度は吉田町=中/戸塚、川島町=旭/保土ケ谷の2件)。
- 南区「平楽」は表に「南事務所にお問合せください。」とだけあり収集曜日非公開のため収録対象外
  (build.mjs の KNOWN_UNPUBLISHED で管理。他の除外行が出たらエラーで停止する)。
- 祝日・大型連休も通常収集 (市 FAQ)。休みは日曜と年末年始のみ。
- 年末年始: 市の告知は毎年 12 月 (令和5年度=12/31〜1/3、令和7年度=12/31〜1/4)。日曜を除いた
  **実効休止は両年とも 12/31〜1/3** で、2026年度もこれを反映 (12/31木・1/1金・1/2土。1/3は日曜)。
  **12月の市告知で要確認・要更新**。
- 日付入り年間カレンダーは市非公開 (ごみ分別アプリ内のみ) のため、日付レベルの独立照合は不可。
- NFKC 正規化で波ダッシュ (〜) がチルダになるため町名表記用に戻している (山元町2〜5丁目 等)。
