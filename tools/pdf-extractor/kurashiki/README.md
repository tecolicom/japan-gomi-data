# 倉敷市 (kurashiki) ごみ収集日 抽出パイプライン

岡山県倉敷市 (code 33202)。市公式「各地区収集日一覧」配下の**地区別PDF 6枚**を一次ソースに、
83コースの course YAML を生成する。収集は6つの環境センター管区(倉敷/水島/玉島/児島/船穂/真備)に分かれ、
真備地区のみ分別が細分化されている(西日本豪雨後の再編)。

## 形式

- 一次ソース: 罫線グリッドのPDF表(テキスト層健全)。縦マージセルは pdfplumber の cell 幾何で検出し上方向フィル。
- 粒度: 曜日 + 第n回目(その月 n 回目の該当曜日)。日付入り通年カレンダーは市が非公開のため日付レベル照合は不可。
- 収録単位: PDF の「地区」セルは町名の列挙(境界条件つきフリーテキスト)。これを **1 area = 1 町名
  (丁目単位)** へ機械分解する(`areas.mjs`。規約は下記)。読み(yomi)はデジタル庁 ABR 倉敷市
  町字マスター由来。原文の学区/管区は course_name_ja に文脈として残す。

## 実行

```sh
node fetch.mjs                 # 6 PDF + 検証用 data eye CSV を cache/ へ取得
node fetch-yomi.mjs            # ABR 倉敷市町字マスター → cache/abr-town.json (yomi + machiaza_id ソース)
python3 extract.py             # cache/records.json (地区ゾーンごとの weekly/第n)
EXTRACTED_AT=YYYY-MM-DD node build.mjs   # area 分解 + yomi 付与 → municipalities/okayama/kurashiki/2026/
node verify.mjs                # 自己照合(categoriesOn 通年再展開 == 抽出)
python3 verify_csv.py          # 独立照合(data eye 平成31年度CSVと曜日/第n)
python3 cross_engine.py        # エンジン非依存(pdfplumber vs pdftotext, 真備・船穂)
```

依存: Python `pdfplumber`、`pdftotext`(poppler)、`unzip`(ABR zip 展開)。

## area 分解規約 (`areas.mjs`)

PDF「地区」セルの地名列挙を町名単位へ分解する。**想定外の表記は throw**(黙って落とさない)。
分解の対応表は `cache/area_expansion.json`(元 125 行 → 展開 area)へ出力する。

- **前処理**: 「船穂町 <字>」の空白は大字-字連結(町名の一部)、方角対「東西・/南北・」直後の
  「・」は区切りでなく連結(東西寿町 等)として先に正規化。
- **前置グループラベル**: 文字列先頭の `Ｒ2南/北`・`ＪＲ東西南北`(直後の `（…）` 含む)は
  町名ではなく、その行の全 area の note に前置。
- **トップレベル区切り**: 括弧の外の `・` `，` 半角 `,` で「参照」に分割(括弧内の区切りは保護)。
- **参照の分類**: 数字始まり = 直前の町の継続(丁目 追加 or 番地)。日本語始まり = 新しい町
  (町名 = 先頭の非数字・非空白の連続)。
- **丁目の展開**: `N丁目` `N`(裸,<100) `N～M丁目` `N丁目～M丁目` は個別丁目へ展開し、それぞれ
  独立 area(name=「<町>N丁目」)にする。列挙全体に係る後置括弧条件は全丁目へ、`N丁目<番地>` の
  番地部は当該丁目のみへ付与。
- **note に保持(展開しない)**: 番地/号/番地レンジ(`N番`・`N号`・`N～M`・`N番地`)、道路/河川境界
  (`◯◯線より東`・`以南`)、`◯◯を除く`・`◯◯のみ`・`◯◯の一部`、字・町内会補足、前置ラベル、
  旧呼称(kyu、行の全 area へ継承)。括弧 `（…）` は直前トークンの note。
- **方角対の展開**: `東西X` → `東X`+`西X`、`南北X` → `南X`+`北X`(実在の公式町名)。
- **重複除去**: 同一 (name, note) の反復はコース内で 1 件に畳む。

分解結果: 125 行 → 491 area(重複除去後 474)。管区別 倉敷139/水島125/玉島62/児島145/船穂6/真備14。

## yomi (読み) と machiaza_id (町字ID)

デジタル庁 **アドレス・ベース・レジストリ (ABR)** 倉敷市町字マスター(`mt_town_city332020`、
大字・町名カナ・丁目番号・machiaza_id)由来。ベース町名+丁目番号で照合する。ABR の町名は
管区/大字接頭辞つき(玉島服部・児島唐琴・水島高砂町・真備町箭田・茶屋町早沖・連島町亀島新田)や
町 サフィクス(老松→老松町)、`通り→通`・`の→之` の表記ゆれが多いため、管区ごとの接頭辞候補と
異形を試して照合し、丁目番号まで一致する町字行の `machiaza_id`(`332020-<7桁>`)と yomi を引く
(丁目行が無ければ大字行)。**接頭辞ぶんの読みは表示名に合わせて除去**(玉島服部=たましまはっとり
→ name 服部 / yomi はっとり。岡山の「岩井1丁目→いわい」と同じく文脈接辞は yomi に含めない)。
- **yomi**: 引けない町名(町内会・字・未登録埋立地名等)は **無し=推測しない**。付与率 **95.1%**
  (展開 491 中 ABR 466・ひらがな自明 1・未付与 24)。未付与は 緑の里・別府・観音台・城・美野団地・
  金浜 等の町内会/字級。
- **machiaza_id**: ABR 行が一意に定まる時のみ付与(付与率 **93.5%**、459/491)。丁目無しで参照された
  丁目登録町(天城台・中央 等)は複数丁目に該当し ID を一意化できないため yomi のみ(ID 無し)。

## note の方針

note は PDF「地区」セル由来の**地理的区別子・限定条件のみ**(番地/道路河川境界/「◯◯を除く」/旧呼称/
字・町内会補足)。岡山で導入された案内文除去(「朝7時30分までに…」「〜ご確認ください」等)の規約は、
倉敷の一次ソースにそうした運用文が含まれないため **該当ゼロ(除去対象なし)**。将来 note に案内文が
混入する表記が現れたら岡山 `stripTimeNote` 同型の除去を追加すること。

## 注意

- `tamashimafunao.pdf` は玉島+船穂を同梱。build/extract では玉島を同PDFから、船穂は `funao.pdf` から取り(重複回避)。
- `cross_engine.py` の値トークン判定は地名に含まれる曜日漢字(「柳井原・水江」の水 等)を除外するため、
  数字・曜日・区切りのみのトークンに限定している。
- 年末年始・祝日運用は一次ソースに実日付の明示が無い(真備は12月広報告知型)。override は作らない。
- yomi ソースの ABR 町字マスターは政府標準利用規約(CC BY 4.0 互換)。CSV は事実データの読み参照に使用。
  ABR に無い町名(町内会・字・未登録埋立地名)は yomi を付けない(推測でカナを作らない)。
- キャッシュ(`cache/`。PDF・CSV・ABR zip/CSV・records.json・abr-town.json・area_expansion.json)は
  成果物ではないので `.gitignore` 追加を推奨(統括者へ報告)。`area_expansion.json` は分解の対応表(監査用)。
