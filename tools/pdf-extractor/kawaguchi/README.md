# 川口市 ごみ収集カレンダー 抽出パイプライン

埼玉県川口市 (handle: `kawaguchi`, code 11203) の地区別収集カレンダー PDF 群 (18地区) から
course YAML を生成する。一次ソースは市公式サイトの地区別 PDF (日付入り通年カレンダー・テキスト層あり)。

## ソース

- 案内: https://www.city.kawaguchi.lg.jp/soshiki/01100/040/4/2/3488.html
- 本体: `material/files/group/94/2026---{1..18}.pdf` — 地区①〜⑱の A3・2ページ (1ページ=6ヶ月)。
  **各 PDF ヘッダに市が収集ルールを 5 行で明示印字する** (所沢と違い日付から規則を推定しない):
  - `● 一般・有害ごみ  X・Y曜日` (weekly 2曜日)
  - `▲ プラスチック製容器包装  水曜日` (全地区 weekly 水)
  - `びん・飲料かん  第n・m曜日` (monthly_nth)
  - `金属類・紙類  第n・m曜日` (monthly_nth)
  - `ペットボトル・繊維類  第n・m曜日` (monthly_nth)
- 番号表: `material/files/group/94/banngouitirannhyou.pdf` — 住所→カレンダー番号 (地区↔町丁目)。
- カレンダーは**暦年 2026 (1〜12月)**。年度ではない。

## パイプライン

```sh
node fetch.mjs                          # 18 地区 PDF + 番号表を cache/pdf/ へ
python3 parse.py                        # ヘッダ規則 + 本体グリッド抽出 & 二表現照合レポート
                                        #   -> cache/rules.json / grid.json / expected.json / areas.json
EXTRACTED_AT=2026-07-20 node build.mjs  # rules + 年末年始 override + course_name_ja を emit、grid と全日再照合
```

- `parse.py`: (1) ヘッダの 5 ルール行を座標クラスタで読む。(2) カレンダー本体を各月の曜日列で
  行アンカー復元 (曜日列を 2026 暦と突合して検証) し、記号 (●=一般+有害 / ▲=プラ) と資源ラベル
  (びん=びん+飲料かん / 金属=金属+紙 / ペット=ペット+繊維) を各実日付へ割付。(3) 番号表 3列組から
  地区→町丁目を復元。ヘッダ規則の 2026 暦年展開と本体グリッドを全日照合する。
- `build.mjs`: ヘッダ規則を rules へ。年末年始休止 (本体グリッドが空欄=収集なしの実日付) を cancelled
  override に。町丁目は course_name_ja に収録 (構造化 areas は yomi ソース確保後、下記参照)。
- 種別→正典語彙 (同日収集の対を分けて表現):
  一般ごみ=burnable / 有害ごみ=hazardous / プラスチック製容器包装=plastic /
  びん=glass_bottle・飲料かん=beverage_can / 金属類=metal・紙類=paper /
  ペットボトル=pet_bottle・繊維類=paper_cloth。語彙追加なし。

## 検証 (2026-07-20)

- **二表現照合**: 同一 PDF 内の「ヘッダ明示ルール」と「本体カレンダーの実日付グリッド」を独立に抽出し、
  18地区 × 2026暦年 (365日) で全日照合。年末年始休止 (12/29〜12/31, 1/1〜1/2) を除き**相違ゼロ**・
  grid-only (規則が取りこぼす収集) **ゼロ**。本体グリッドの行アンカーは全12ヶ月の曜日列を 2026 暦と
  突合して検証済。
- **自己照合**: 生成 course を categoriesOn で 2026暦年に再展開し本体グリッドと全日一致 (相違ゼロ)。
- **公式検索サイト スポットチェック**: `kawaguchi-gomimaru.jp` の地区別カレンダー (別レンダリング) を
  地区1 (金山町/カレンダー番号1) と地区17 (鳩ヶ谷緑町/カレンダー番号17) で照合。曜日・第n規則・
  同日収集の対 (一般+有害, びん+かん, 金属+紙, ペット+繊維) がいずれも一致。同サイトは同一 PDF 由来のため
  第2の権威ソースには当たらないが、独立系のレンダリングでの一致を確認。

## 出力

`municipalities/saitama/kawaguchi/2026/course-{1..18}.yaml` (18コース = 市公式カレンダー番号①〜⑱に 1:1)。
`cache/` は再生成可能なため非追跡 (.gitignore)。

## 読み (yomi)

`yomi.yaml` = ベース町名→読み(ひらがな)。日本郵便 郵便番号検索 (川口市 11203) の郵便カナ
(`data-addr_kana`) 由来 113 町 + 通例読み 2 件。build.mjs が各 area 名からベース名 (丁目・番地・
（…注記）除去) を引き、未収録なら throw (推測で埋めない)。町丁目 133 件中 131 件が郵便カナで確定、
コンフォール東/西鳩ヶ谷 (郵便町名でない UR 団地) の 2 件のみ通例読み「こんふぉーる…」(要確認)。
