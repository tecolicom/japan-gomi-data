# tokyo-nakano (中野区) CSV 抽出パイプライン

区オープンデータ CSV「資源とごみの収集曜日一覧」(CC BY 4.0) を一次ソースに、
`municipalities/tokyo/tokyo-nakano/2026/course-*.yaml` を生成する。

```
node fetch.mjs        # CSV + 公式HTML + 地域別ページ + カレンダーPDF 42枚 → cache/
node --test           # パーサ単体テスト
node build.mjs        # cache/ → course-*.yaml (42行 → シグネチャ畳み込みで25コース)
python3 verify.py     # 生成 YAML を町丁目別カレンダー PDF 全42枚と通年機械照合
```

## ソースと照合の構造

- **一次ソース**: オープンデータ CSV (東京都カタログ / 実体は中野区データマップ wagmap.jp)。
  最終確認日 2021-02-08 と古いため、build.mjs が**現行公式 HTML 表と全行照合**し、
  不一致なら中断する (鮮度ガード)。
- **検証ソース**: 町丁目別「資源とごみのカレンダー」PDF (`R8-<N>.pdf`)。
  **N は CSV の NO 列と一致**する (2026-07-16 確認)。build.mjs が `cache/no-to-area.json`
  に対応表を出力し、verify.py が全 42 町丁目 × 通年で照合する。

## PDF の罠 (pdf_calendar.py が吸収)

- 非収集セルにも**不可視のラベル文字**が埋まっている (白矩形で隠されているだけ)。
  テキスト層だけを読むと幻の収集日を拾う。**色付き矩形 (チップ) のみを可視と判定**する。
- ページ最下部の凡例チップが最下段の月ブロックに混入する → 「集積所」マーカーより下を除外。
- 「び缶ペ」は PDF によってひらがな「び缶ぺ」表記 (R8-25 等)。
- **区 PDF 側の誤植が実在** (2026年度版): R8-28 は 2027-03 全体が 1 日ズレ印字・
  2027-01 最終週の日付欠落、R8-29 は 5/28・8/27 のチップ文字が「資源プラ」誤記
  (チップ色は陶ガラ金で正)。日付はセル数字でなく**グリッド計算 (週行×曜日列) を正**とする。
- 年末年始: PDF は 12/28 以降の最終週を空白にする (確定告知は 12 月頃)。
  verify.py はこの窓 (12/27〜1/3) を日程比較から除外する。詳細は meta.yaml notes。
