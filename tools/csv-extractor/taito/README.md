# taito (台東区) CSV 抽出パイプライン

区オープンデータ CSV「地域別収集曜日一覧」(CC BY 4.0) を一次ソースに、
`municipalities/tokyo/taito/2026/course-*.yaml` を生成する。

```
node fetch.mjs        # CSV + 公式HTML表 + カレンダー案内 + プラスチック案内 → cache/
node --test           # パーサ単体テスト (8件)
node build.mjs        # cache/ → course-*.yaml (39行108町丁 → 12コース)
```

`build.mjs` は独立3ソースの突合をすべて通らないと出力しない (照合は build に内蔵。
別途 verify スクリプトを持たないのは、粒度が曜日規則 (weekday-rules) であり
日付レベルの独立表現が機械可読な形で存在しないため)。

## ソースと照合の構造

| # | ソース | 役割 |
|---|---|---|
| 1 | オープンデータ CSV (Shift_JIS, 39行) | **一次**。令和4年3月公開と古い |
| 2 | 公式 HTML 表「収集曜日（全体）」 | **鮮度ガード**。現行。全108町丁を突合 |
| 3 | 令和8年度カレンダー案内の「整理番号」1〜12 | **畳み込みの検証**。区公式のコース分け |

- **(2) は CSV の鮮度ガード**: CSV の公開が令和4年と古いため、現行 HTML 表と
  丁目単位で全数突合し、1件でも食い違えば中断する。2026-07-20 時点で不一致0件。
- **(3) は畳み込みの検証**: 日程シグネチャで畳み込んだ 12 グループが、区が地域別
  カレンダー PDF に振っている整理番号 1〜12 と 1 対 1 で完全対応することを確認する。
  **course 番号にはこの整理番号をそのまま採用**している (区の呼称と一致させるため)。

## 表記の罠 (parse.mjs が吸収)

- **CSV は丁目を全列挙** 「浅草1丁目・浅草2丁目」、**HTML は範囲表記** 「浅草3から7丁目」。
  両者を丁目単位の集合へ正規化してから突合する。
- **根岸の行だけ区切りが読点**「根岸1丁目、根岸2丁目・…」(区側の入力ゆれ)。中黒と読点の両方を区切り扱い。
- 丁目を持たない町丁がある: 秋葉原・上野公園。
- **「その月の1回目・3回目の土曜日」は第n週ではない**。月内の該当曜日の出現回数基準
  = `monthly_nth`。区の収集曜日一覧ページも「その月の1回目･3回目 の…日」と表現している。
- CSV は **Shift_JIS**。fetch.mjs が UTF-8 化して保存し、`�` が出たら中断する。

## カテゴリ対応

CSV の列は 資源 / 燃やすごみ / 燃やさないごみ の3つだが、正典語彙へは7カテゴリに展開する。

- `資源` → **paper / glass_bottle / beverage_can / pet_bottle** (集積所で同日回収の総称)
- `燃やすごみ` → burnable、`燃やさないごみ` → non_burnable (月2回 = monthly_nth)
- **plastic は CSV/HTML に列が無い**。令和7年4月から区内全域で分別回収が始まり、
  区公式 (`shigenjoho/plastics.html`) が「資源の曜日（当日朝8時まで）に出してください」と
  明記しているため、**資源と同一の曜日規則**で追加している。CSV/HTML表はプラスチック開始前の
  様式のままなので、**列が無いことを「収集が無い」と読まないこと**。

資源系5種は同じ `days` 配列インスタンスを共有するため、YAML では anchor (`&a1` / `*a1`) に
なり同日収集であることが出力上も明示される。

## 未反映事項 (推測で作らないもの)

- **年末年始**: 区の令和8年度カレンダーが「※年末年始は収集日を変更する場合があります。
  12月の広報たいとうや台東区ホームページなどでご確認ください」と明示し**日程未確定**。
  overrides は置いていない。12月に meta.yaml の `yearend_url` から確認して追記する。
- **燃やさないごみの臨時収集日**: カレンダー PDF に「〇は燃やさないごみ臨時収集日」の凡例が
  あるが、該当日は PDF 上で色チップ (図形) としてのみ表現されテキスト層から抽出できなかった。
- **祝日**: 区公式に明示的記載を見つけられず。カレンダーは例外として年末年始のみ注記している
  (=祝日は通常収集と読める) が、明文の根拠が無いため overrides は置いていない。
