# arakawa (荒川区) CSV 抽出パイプライン

区が配布する収集日 CSV (`gomi_<YYYYMMDD>.csv`, cp932) を一次ソースに、
`municipalities/tokyo/arakawa/2026/course-*.yaml` を生成する。

```
node fetch.mjs          # ポータルから CSV + 公式HTML → cache/ (UTF-8 へ変換して保存)
node --test             # パーサ単体テスト (13 件)
node build.mjs          # cache/ → course-*.yaml (2,249行 → 129コース / 514 area)
python3 verify.py       # 独立実装(Python)で CSV ⇔ 生成YAML を全行・通年照合
node verify-html.mjs    # 区公式HTML表 (独立表現) と ごみ3種別を照合
```

`cache/` は生成物なのでコミットしない (リポジトリ `.gitignore` に
`tools/csv-extractor/arakawa/cache/` の追記が必要 — 統合担当へ依頼済み)。

## ソースの構造

| | 内容 | 粒度 | 備考 |
|---|---|---|---|
| 一次 | 配布 CSV | 地区×丁目×番地・号 (2,249行) | 全8種別。cp932。更新 2025-12-16 |
| 検証 | 公式「ごみ収集日・プラスチック回収日」HTML | 住所レンジ (46行) + 物件名 (30件) | **ごみ3種別のみ**。更新 2026-06-29 |

HTML 側のほうが更新日が新しいため、`verify-html.mjs` は照合であると同時に
**CSV の鮮度ガード**にもなっている (CSV が古びれば ごみ3種別に不一致が出る)。

CSV の列 6 種 → 正典語彙 8 種の対応は `municipalities/tokyo/arakawa/taxonomy.yaml` を参照。
「びん・缶、古紙」は 1 列 = 3 種別の同日収集なので days 配列を共有 (YAML anchor) して同日性を明示する。

## area 名の規約

CSV は番地単位で行数が多い (2,249 行) ため、同一日程の行を 1 コースへ畳んだうえで
area 名を次の規則で圧縮する (可読性優先):

1. **丁目が丸ごと 1 コース**に収まるなら丁目名だけ — `西尾久4丁目`
2. そうでなければ**連続する番地をレンジに畳む** — `南千住1丁目1〜13番`
3. 連続しない番地は個別に列挙 — `南千住1丁目16番`
4. **「号」まで含む行は畳まずそのまま**残す — `荒川7丁目9番12号`、`南千住7丁目20番6号～`

`yomi` は五十音ソート用に `<地区の読み><丁目番号>-<開始番地>` (例 `みなみせんじゅ1-16`)。
地区の読みは `yomi.yaml` (日本郵便の郵便番号データで確認)。

この畳み込みが情報を落としていないことは `verify.py` が
**area 名を (地区,丁目,番地・号) へ逆写像し、CSV 全行を重複0・漏れ0で被覆すること**を確認して保証する。

## 表記の罠

- **「第n」の表記ゆれ**: `第1・第3の金曜日` / `第1・第3・第5の木曜日` / `第2・第4の木` (「曜日」省略)
  / `第4の月曜日`。`parse.mjs` が `第`除去 + `の`除去で `_lib` の `parseMonthlyNthJa` が読む形へ正規化する。
- **複合表記が 1 行だけ実在**: 荒川7丁目9番12号 の
  `古紙：第2・第4の土曜日、びん缶：月曜日` — 古紙だけ別日。`parseBinKanPaper` が分割する。
- **展開できないセル 271 個**: `個別` (117, 大規模集合住宅の個別回収) / `お問い合わせください` (66)
  / `不定期` (4) / 古布の空欄 (64)。いずれも資源列のみで、ごみ3種別は全行確定している。
  該当カテゴリは rules から落とす (無音では落とさず build のログに件数を出す)。
  未知の表記は `throw` する。
- **年末年始はごみと資源で休止幅が違う**: ごみ・プラは 1/1〜1/3、資源は **12/31** 〜1/3。
  CSV の備考欄 2 列にそれぞれ明示されており、`build.mjs` は文言が変わると throw する。
- **CSV に重複行が 1 件**: `西尾久8丁目44番` が 2 行あり資源の日程が食い違う。隣接番地との整合から
  1 行目を採用 (先勝ち)。`KNOWN_DUPLICATES` に登録済みで、未知の重複が現れると build が落ちる。

## 検証で残った未解決

`verify-html.mjs` は HTML 表と一致しない行を「大規模集合住宅の別表で説明できるか」で二段判定する。
2026-07-20 時点: 2,220 行中 2,209 一致 / 9 行は別表と一致 / **2 行が未解決**
(`南千住3丁目41番3~5号`・`41番6号`)。別表 30 物件が汐入地区を網羅していないためと推定。
一次ソースを正として出力し、`KNOWN_UNEXPLAINED` に登録している (増えたら exit 1)。

資源5種別を載せた独立表現は区サイトに見当たらなかった (**資源側の第2表現は無し**)。
地域別カレンダー PDF (`/documents/735/…`) は存在するので、将来的な照合先の候補。

## 来年度更新

1. `node fetch.mjs` — ポータルの CSV リンクを自動追跡するので、新版が出ていれば取得できる
   (URL が変わると警告を出す。`build.mjs` の `CSV_URL` は `cache/source-url.txt` から読むので追随する)
2. `build.mjs` の `YEAR` / `fiscal_year_ja` / `STOPPED_ON` (年末年始の実日付) を更新
3. fetch → `node --test` → build → `verify.py` → `verify-html.mjs` を再実行
4. 12/31 の曜日が変わり「同じ日にごみと資源が両方ある」コースが生じた場合、
   `build.mjs` はカテゴリ別 `cancelled` override を出す。正典実装 `tools/_lib/schedule.mjs` の
   `categoriesOn()` は同日に cancelled が 1 件でもあると全カテゴリを落とす実装なので、
   その年度では正典側の解釈を統合担当と確認すること (2026年度は該当 0 件で影響なし)。
